Why GstAdaptNode?

Hardware Auto-Detection

Probes /dev/ for NVIDIA Jetson or Intel VA-API at startup, then live-validates against the GStreamer registry. Broken accelerators (like vaapipostproc on 1.20) are skipped, not attempted.

Zero-Copy Hot Path

Intra-process unique_ptr move into the subscriber, then a custom appsrc wrapper moves the pointer straight into GStreamer via gst_buffer_new_wrapped_full. No memcpy on the hot path.

Dual-Mode Fallback

When no usable GPU path is present, the node bypasses GStreamer entirely and runs cv::resize in the subscriber callback. Faster than image_proc even without a GPU because there's no image_transport round-trip.

The Plug-and-Play Promise

Same parameters. Same composable-node interface. Your camera still publishes bgr8. Toggle below to see the difference.

CPU-bound — image_proc 
ComposableNode(
    package='image_proc',
    plugin='image_proc::ResizeNode',
    name='resize',
    parameters=[{
        'use_scale': False,
        'height': 480,
        'width': 640,
    }],
)

ComposableNode(
    package='gst_adapt_node',
    plugin='gst_adapt_node::ResizeNode',
    name='resize',
    parameters=[{
        'use_scale': False,
        'height': 480,
        'width': 640,
    }],
)

Two strings change. Everything else stays the same.

Quick Start

# Clone
cd ~/ros2_ws/src
git clone --recursive https://github.com/sohams25/GstAdaptNode.git gst_adapt_node

# Dependencies
sudo apt install ros-humble-image-proc gstreamer1.0-vaapi \
  libgstreamer1.0-dev libgstreamer-plugins-base1.0-dev
pip install flask psutil

# Build
cd ~/ros2_ws && colcon build --packages-up-to gst_adapt_node
source install/setup.bash

# A/B stress test (any 4K mp4)
ros2 launch gst_adapt_node A_B_comparison.launch.py video_path:=/path/to/4k.mp4

# Dashboard (second terminal)
ros2 run gst_adapt_node visualize_demo.py
# → open http://localhost:8080

The launch file purges stale Fast-DDS /dev/shm locks, kills orphan containers from prior runs, and pins a UDPv4-only Fast-DDS profile to silence RTPS_TRANSPORT_SHM init errors.
