The Plug-and-Play Promise

gst_adapt_node::ResizeNode is a drop-in replacement for image_proc::ResizeNode. Same parameters. Same composable node interface. Change two strings.

Before — CPU-bound 
ComposableNode(
    package='image_proc',
    plugin='image_proc::ResizeNode',
    name='resize',
    parameters=[{
        'use_scale': False,
        'height': 480,
        'width': 640,
    }],
)
After — Hardware-Accelerated 
ComposableNode(
    package='gst_adapt_node',
    plugin='gst_adapt_node::ResizeNode',
    name='resize',
    parameters=[{
        'use_scale': False,
        'height': 480,
        'width': 640,
    }],
)

Your camera still publishes bgr8. Your perception stack still receives bgr8. Everything in between runs on the GPU.

How It Works

01

Detect Hardware

Probes /dev/nvhost-* for Jetson and /dev/dri/renderD* for Intel. No config files. No environment variables.

02

Validate Plugins

Queries the GStreamer registry for vaapipostproc or nvvideoconvert. Falls back to CPU with a visible warning if missing.

03

Accelerate

Builds a hardware-specific pipeline at runtime. Custom appsrc/appsink wrapper feeds BGR frames directly to the GPU — zero copies.

Zero-Copy Data Path

Native bgr8 ingestion. No format conversion required upstream. The GPU handles color conversion + resize in a single pass.

Camera / MediaStreamerNode (bgr8, unique_ptr)
↓ intra-process pointer move — 0 copies
ResizeNode::on_image()
↓ gst_buffer_new_wrapped_full — 0 copies
appsrc → videoconvert → vaapipostproc → vaapipostproc → videoconvert → appsink
‹ GPU resize via VA-API / NVMM hardware ›
↓ on_new_sample() publishes processed bgr8
output_topic (bgr8, 640×480)

Quick Start

# Clone into a colcon workspace
cd ~/ros2_ws/src
git clone --recursive https://github.com/sohams25/GstAdaptNode.git gst_adapt_node

# Install dependencies
sudo apt install ros-humble-image-proc gstreamer1.0-vaapi \
  libgstreamer1.0-dev libgstreamer-plugins-base1.0-dev

# Build
cd ~/ros2_ws && colcon build --packages-up-to gst_adapt_node
source install/setup.bash

# Run the A/B stress test
ros2 launch gst_adapt_node A_B_comparison.launch.py video_path:=/path/to/4k.mp4

# Visual dashboard (second terminal)
ros2 run gst_adapt_node visualize_demo.py
